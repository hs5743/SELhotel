# SEL 情緒大飯店

SEL 情緒大飯店是一個結合 SEL（社會情緒學習）與班級經營的互動網站。學生可以選擇班級飯店、輸入房卡密碼，替自己的心情辦理入住；老師則能建立飯店、觀察班級情緒狀態，並在需要時管理不適當內容。

網站目前部署於 GitHub Pages，資料儲存在 Firebase Firestore。整體設計目標是讓情緒表達變得具體、溫和、可被看見，也讓同儕回饋從抽象的「關心」變成可以練習的文字行動。

## 網站主要功能概要

- **建立班級情緒飯店**：教師可建立不同班級或活動使用的飯店，並設定房卡密碼。
- **學生 Check-in**：學生選擇飯店、輸入密碼後，填寫姓名、心情、房門留言、房間顏色、裝飾與樓層。
- **房間牆展示**：所有入住者會以房間卡片呈現在同一面房間牆上，依樓層排列。
- **心情天氣台**：系統依學生選擇的心情分類，自動統計班級目前的情緒分布。
- **溫暖信箱**：使用者可進入他人的房間，留下支持、理解或回應的留言。
- **教師後台**：教師可使用 Google 帳號登入，管理飯店、重設房卡密碼、刪除個別房間或不適當留言。
- **展示型主題飯店**：可預先匯入主題角色與對話，用於研習、課堂展示或 SEL 活動示範。

## 網站發展歷程概要

1. **Google Apps Script + Google Sheets 原型**
   - 最初版本使用 Google Apps Script 製作前端介面，資料存放於 Google Sheets。
   - 已具備建立飯店、學生入住、房間牆、回饋留言與管理刪除等核心概念。

2. **GitHub Pages + Firebase 移植**
   - 將網站改為靜態前端部署於 GitHub Pages。
   - 使用 Firebase Firestore 儲存飯店、房間與留言資料。
   - 使用 Firebase Authentication 的 Google 登入作為教師管理權限基礎。

3. **v1 穩定版**
   - 完成 Firebase 資料讀寫、飯店密碼登入、學生入住、房間即時顯示、樓層排序與回饋留言功能。
   - 支援樓層範圍 `1000F` 到 `B18`。
   - 標記為可回復的穩定版本。

4. **v2 視覺升級**
   - 網站改為暖光飯店風格，加入品牌化 logo、首頁主視覺與飯店房間風格圖。
   - 首頁文案調整為 `學生 Check-in`、`教師開店`、`房卡入住`、`心情天氣`、`溫暖信箱`。
   - 保留原有 Firebase 資料結構與功能流程。

5. **管理功能擴充**
   - 教師後台新增飯店管理入口。
   - 可重設飯店房卡密碼。
   - 可刪除個別入住者，並同步刪除該房間所有留言。
   - 可刪除個別留言，適合處理不當暱稱或攻擊性內容。

6. **教學展示資料擴充**
   - 建立多組展示飯店，協助說明網站功能與 SEL 應用情境。
   - 曾匯入歷史學習取向的主題飯店，以及較貼近日常生活的「人生轉運站大飯店」。

## 網站主要功能介紹

### 1. 首頁與飯店入口

首頁提供兩個主要入口：

- **學生 Check-in**：學生選擇想入住的飯店，輸入房卡密碼後進入情緒大廳。
- **教師開店**：教師建立新的班級飯店，設定飯店名稱、房卡密碼與視覺主題。

首頁也會顯示目前營運中的飯店數量，方便教師確認資料是否已載入。

### 2. 飯店大廳

進入飯店後，左側會顯示：

- 飯店名稱
- 飯店風格圖片
- 入住人數
- 主要情緒
- 辦理入住按鈕
- 離開大廳按鈕

這個區域可讓教師快速掌握班級或活動目前的整體狀態。

### 3. 辦理入住

學生入住時可以填寫：

- 名字或暱稱
- 目前心情
- 房門留言
- 房間主色
- 房間裝飾
- 想住的樓層

心情分為四大類：

- 愉快好心情
- 靜靜好舒服
- 緊張小心臟
- 心情灰灰的

房門留言目前支援最多 500 字，適合學生寫下比較完整的心情描述或角色扮演內容。

### 4. 房間牆

房間牆會把所有入住者以房間卡片呈現，並依樓層排序。每張房間卡會顯示：

- 入住者名稱
- 心情詞
- 房間顏色
- 房間裝飾
- 回饋留言數

點擊房間後，可以開啟房間彈窗，閱讀房門留言與溫暖信箱內容。

### 5. 心情天氣台

心情天氣台會依入住者選擇的心情分類，自動計算各類心情的人數與比例。  
教師可用它觀察班級目前的情緒氣候，例如：班上是否偏焦慮、疲累，或是否多數人感到平靜、有希望。

### 6. 溫暖信箱

每間房都有溫暖信箱，其他使用者可以留下回饋留言。  
回饋留言目前支援最多 250 字，適合寫下支持、理解、提醒、鼓勵或角色互動對話。

這個功能可用於：

- 同儕支持練習
- 班級回饋活動
- 角色觀點對話
- 歷史人物或情境人物的互動展示

### 7. 教師後台

教師可透過 Google 帳號登入後台。後台功能包含：

- 查看所有飯店
- 重設飯店房卡密碼
- 進入單一飯店管理頁
- 刪除不適當入住者
- 刪除個別不適當留言
- 刪除整間飯店

這些功能讓教師能維持教學現場的安全感與秩序，同時避免因少數不當內容影響整體活動。

## Firebase 資料結構

```text
hotels/{hotelId}
hotels/{hotelId}/access/{passcodeHash}
hotels/{hotelId}/rooms/{roomId}
hotels/{hotelId}/rooms/{roomId}/feedbacks/{feedbackId}
```

主要資料分工：

- `hotels`：飯店名稱、主題、房間數、是否封存。
- `access`：飯店房卡密碼的雜湊值。
- `rooms`：入住者、心情、房門留言、房色、裝飾、樓層、留言數。
- `feedbacks`：單一房間中的溫暖信箱留言。

## 部署與維護

- 前端部署：GitHub Pages
- 資料庫：Firebase Firestore
- 教師登入：Firebase Authentication / Google Sign-In
- 安全規則：`firestore.rules`

若更新網站程式，推送 `main` branch 後由 GitHub Pages 發布。  
若更新 Firestore 權限，需另外部署 Firebase security rules。

## 版本回復

若需要回到已標記的穩定版本，可使用 Git tag 或 revert 指令處理。

```powershell
git checkout v1
```

若只是取消某一次視覺或功能提交，建議使用：

```powershell
git revert <commit-sha>
git push origin main
```
